# 03. Конструкторы и деструкторы

В прошлом уроке мы создали класс `Animal` с конструктором. Теперь разберём конструкторы подробнее и познакомимся с деструкторами.

## Конструктор по умолчанию

В уроке 02 мы писали:

```cpp
Animal dog("Bobik", 3);
```

А что если хочется создать объект без параметров — заполнить данные потом?

```cpp
Animal dog; // ошибка компиляции!
```

Не работает. Почему? Когда мы написали конструктор с параметрами, компилятор перестал создавать конструктор без параметров автоматически. Теперь он говорит: "ты сам решил, как создавать объекты — вот и создавай только так".

Решение — добавить **конструктор по умолчанию** (без параметров):

```cpp
class Weapon {
private:
    std::string name;
    int damage;

public:
    // конструктор по умолчанию
    Weapon() {
        name = "Fist";
        damage = 5;
    }

    // конструктор с параметрами
    Weapon(std::string n, int d) {
        name = n;
        damage = d;
    }

    void print() {
        std::cout << name << " (урон: " << damage << ")\n";
    }
};
```

Теперь можно и так, и так:

```cpp
Weapon fist;                    // Fist, урон 5
Weapon sword("Sword", 25);     // Sword, урон 25
```

## Инициализация полей в объявлении

Вместо того чтобы присваивать значения в конструкторе по умолчанию, можно задать их прямо при объявлении полей:

```cpp
class Weapon {
private:
    std::string name = "Fist";
    int damage = 5;

public:
    Weapon() {}  // тело пустое — поля уже инициализированы

    Weapon(std::string n, int d) {
        name = n;
        damage = d;
    }
};
```

Если конструктор не трогает поле — оно получит значение из объявления. Если трогает — перезапишет.

## Список инициализации

Посмотри на конструктор:

```cpp
Weapon(std::string n, int d) {
    name = n;
    damage = d;
}
```

Здесь происходит два шага: сначала `name` создаётся пустой строкой (значение по умолчанию), потом ей присваивается `n`. Для строк это лишняя работа — создали пустую, тут же выбросили и записали другую.

Есть способ лучше — **список инициализации**. Он пишется через двоеточие после скобок, до тела конструктора:

```cpp
Weapon(std::string n, int d) : name(n), damage(d) {
    // тело пустое — всё уже задано
}
```

Зачем? Присваивание в теле конструктора — это два шага: сначала поле создаётся с пустым значением, потом ему присваивается новое. Список инициализации — один шаг: поле сразу создаётся с нужным значением.

Для `int` разница незаметна. Для строк и сложных объектов — может быть ощутима.

Правило простое: если можешь использовать список инициализации — используй.

## Деструктор

Мы научились создавать объекты. Но что происходит, когда объект больше не нужен? Например, функция закончилась — и все локальные переменные исчезают. Если объект открыл файл или выделил память — кто это почистит?

Для этого есть **деструктор** — метод, который вызывается автоматически при уничтожении объекта.

Когда объект уничтожается? Когда заканчивается область видимости, в которой он объявлен:

```cpp
void battle() {
    Weapon sword("Sword", 25);  // конструктор
    // ... бой ...
}  // здесь sword уничтожается — вызывается деструктор
```

Деструктор пишется как конструктор, но с тильдой `~` перед именем:

```cpp
class Weapon {
private:
    std::string name;
    int damage;

public:
    Weapon(std::string n, int d) : name(n), damage(d) {
        std::cout << name << " создан\n";
    }

    ~Weapon() {
        std::cout << name << " уничтожен\n";
    }

    void print() {
        std::cout << name << " (урон: " << damage << ")\n";
    }
};
```

Деструктор не принимает параметров и ничего не возвращает. У класса может быть только один деструктор.

## Полный пример

```cpp
#include <iostream>
#include <string>
#include <Windows.h>

class Weapon {
private:
    std::string name = "Fist";
    int damage = 5;

public:
    Weapon() {
        std::cout << "Создано оружие по умолчанию: " << name << "\n";
    }

    Weapon(std::string n, int d) : name(n), damage(d) {
        std::cout << "Создано оружие: " << name << "\n";
    }

    ~Weapon() {
        std::cout << "Оружие " << name << " уничтожено\n";
    }

    void print() {
        std::cout << name << " (урон: " << damage << ")\n";
    }
};

int main() {
    SetConsoleCP(1251);
    SetConsoleOutputCP(1251);

    Weapon fist;
    fist.print();

    Weapon sword("Sword", 25);
    sword.print();

    std::cout << "\n--- конец main ---\n";
    return 0;
}
```

```text
Создано оружие по умолчанию: Fist
Fist (урон: 5)
Создано оружие: Sword
Sword (урон: 25)

--- конец main ---
Оружие Sword уничтожено
Оружие Fist уничтожено
```

Обрати внимание: объекты уничтожаются в обратном порядке. Первым создан `fist` — последним уничтожен. Это как стопка тарелок: последнюю поставил — первую снял.

## Зачем нужен деструктор

Сейчас деструктор просто выводит текст — не очень полезно. Но в реальных программах деструктор:
- освобождает выделенную память
- закрывает файлы
- отключается от сети

Если класс не выделяет ресурсов — деструктор можно не писать. Компилятор создаст пустой автоматически.

## Шпаргалка

- Конструктор по умолчанию — без параметров: `Weapon() {}`
- Конструктор с параметрами — задаёт начальные значения: `Weapon(string n, int d)`
- Можно иметь оба конструктора одновременно
- Инициализация полей в объявлении: `int damage = 5;`
- Список инициализации: `Weapon(string n, int d) : name(n), damage(d) {}`
- Деструктор: `~Weapon()` — вызывается при уничтожении объекта
- Объекты уничтожаются в обратном порядке создания

```cpp
class Weapon {
private:
    std::string name = "Fist";
    int damage = 5;

public:
    Weapon() {}
    Weapon(std::string n, int d) : name(n), damage(d) {}
    ~Weapon() {}
    void print() { std::cout << name << " (урон: " << damage << ")\n"; }
};
```

## Практика

### 3.1. Зелья

Создай класс `Potion` с полями: название (`name`) и сила действия (`power`).

Два конструктора:
- по умолчанию: `"Small Heal"`, сила 20
- с параметрами: любые значения

Деструктор выводит: `"Зелье <name> использовано"`.

Проверь:

```cpp
Potion small;
small.print();

Potion big("Mega Heal", 100);
big.print();
```

```text
Зелье Small Heal создано
Small Heal (сила: 20)
Зелье Mega Heal создано
Mega Heal (сила: 100)
Зелье Mega Heal использовано
Зелье Small Heal использовано
```

### 3.2. Область видимости

Что выведет эта программа? Сначала подумай, потом проверь.

```cpp
int main() {
    Weapon bow("Bow", 15);
    {
        Weapon dagger("Dagger", 10);
        dagger.print();
    }
    std::cout << "Между блоками\n";
    bow.print();
    return 0;
}
```

Подсказка: фигурные скобки `{}` создают новую область видимости. Когда она заканчивается — объекты внутри уничтожаются.

### 3.3. Персонаж с оружием

Создай класс `Player` с полями: имя, здоровье, название оружия, урон оружия.

Конструктор принимает только имя и здоровье. Оружие по умолчанию — `"Fist"` с уроном 5.

Добавь метод `equip(std::string weaponName, int weaponDamage)`, который меняет оружие.

Проверь:

```cpp
Player hero("Knight", 100);
hero.printStatus();  // Knight: 100 HP, оружие: Fist (5)

hero.equip("Sword", 25);
hero.printStatus();  // Knight: 100 HP, оружие: Sword (25)
```
