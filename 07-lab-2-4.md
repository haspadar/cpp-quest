## Про геттеры и сеттеры в лабе

Лаба требует геттеры и сеттеры. В наших уроках мы говорили: «скажи, не спрашивай», сеттеры — не лучшая практика. Как быть?

Просто: **для лабы пишем то, что просят**. Преподаватель хочет увидеть, что ты знаешь синтаксис. Но в голове держи: в настоящем коде `student.setGrade(5)` лучше заменить на `student.passExam("Математика", 5)`.

Для лабы — сеттер. Для жизни — метод с поведением. Ты знаешь разницу, и это главное.

## Пример: вариант «DateClass»

В методичке показан `DateClass` — класс для хранения даты. Разберём, как бы мы его сделали.

### Версия a) — один файл

```cpp
#include <iostream>
#include <string>
#include <Windows.h>

class DateClass {
private:
    int day = 0;
    int month = 0;
    int year = 0;

public:
    // Конструктор по умолчанию
    DateClass() {
        std::cout << "DateClass создан (по умолчанию)\n";
    }

    // Конструктор с параметрами
    DateClass(int d, int m, int y) : day(d), month(m), year(y) {
        std::cout << "DateClass создан: ";
        print();
    }

    // Деструктор
    ~DateClass() {
        std::cout << "DateClass уничтожен: ";
        print();
    }

    // Сеттер
    void setDate(int d, int m, int y) {
        day = d;
        month = m;
        year = y;
    }

    // Геттеры
    int getDayDate() const { return day; }
    int getMonthDate() const { return month; }
    int getYearDate() const { return year; }

    // Вывод
    void print() const {
        std::cout << day << "." << month << "." << year << "\n";
    }

    // operator<< — это мы умеем, а в методичке этого нет :)
    friend std::ostream& operator<<(std::ostream& out, const DateClass& d) {
        out << d.day << "." << d.month << "." << d.year;
        return out;
    }
};

int main() {
    SetConsoleCP(1251);
    SetConsoleOutputCP(1251);

    // Массив объектов
    DateClass dates[3];
    dates[0].setDate(15, 4, 2026);
    dates[1].setDate(1, 9, 2026);
    dates[2].setDate(31, 12, 2026);

    for (int i = 0; i < 3; i++) {
        std::cout << "Дата " << (i + 1) << ": " << dates[i] << "\n";
    }

    return 0;
}
```

### Версия b) — разделение на файлы

**DateClass.h:**

```cpp
#pragma once
#include <iostream>

class DateClass {
private:
    int day = 0;
    int month = 0;
    int year = 0;

public:
    DateClass();
    DateClass(int d, int m, int y);
    ~DateClass();

    void setDate(int d, int m, int y);
    int getDayDate() const;
    int getMonthDate() const;
    int getYearDate() const;
    void print() const;

    friend std::ostream& operator<<(std::ostream& out, const DateClass& d);
};
```

**DateClass.cpp:**

```cpp
#include "DateClass.h"

DateClass::DateClass() {
    std::cout << "DateClass создан (по умолчанию)\n";
}

DateClass::DateClass(int d, int m, int y) : day(d), month(m), year(y) {
    std::cout << "DateClass создан: ";
    print();
}

DateClass::~DateClass() {
    std::cout << "DateClass уничтожен: ";
    print();
}

void DateClass::setDate(int d, int m, int y) {
    day = d; month = m; year = y;
}

int DateClass::getDayDate() const { return day; }
int DateClass::getMonthDate() const { return month; }
int DateClass::getYearDate() const { return year; }

void DateClass::print() const {
    std::cout << day << "." << month << "." << year << "\n";
}

std::ostream& operator<<(std::ostream& out, const DateClass& d) {
    out << d.day << "." << d.month << "." << d.year;
    return out;
}
```

**main.cpp:**

```cpp
#include <iostream>
#include <Windows.h>
#include "DateClass.h"

int main() {
    SetConsoleCP(1251);
    SetConsoleOutputCP(1251);

    DateClass dates[3];

    dates[2].setDate(31, 12, 2026);

```

## Сравнительный анализ (задание 2 лабы)

Лаба просит заполнить таблицу «структура vs класс». Вот подсказки:

| | Структура (`struct`) | Класс (`class`) |
|---|---|---|
| Доступ по умолчанию | `public` | `private` |
| Защита данных | Нет — любой код может записать что угодно | Есть — private-поля доступны только через методы |
| Контроль значений | Нет — `date.month = 13` пройдёт | Есть — сеттер может проверить: `if (m < 1 \|\| m > 12)` |
| Инициализация | Можно забыть заполнить поле | Конструктор гарантирует начальные значения |
| Поведение | Данные без логики | Данные + методы = объект с поведением |

### 7.1. Твой вариант

Возьми свой вариант из предыдущей лабораторной. Реализуй его как класс:

b) В трёх файлах: `.h`, `.cpp`, `main.cpp`.

### 7.2. Добавь поведение

Подумай: какие **действия** имеют смысл для объектов твоего варианта? Не просто `setX` / `getX`, а осмысленные операции.

Примеры:
- «Студент»: `passExam("Математика", 8)`, `isScholar()` (стипендиат ли)
- «Книга»: `borrow()` (взять), `returnBook()` (вернуть), `isAvailable()`
- «Автомобиль»: `drive(100)` (проехать), `refuel(30)` (заправить), `needsRefuel()`

Добавь 2-3 таких метода. Это не требуется в лабе, но покажет преподавателю, что ты понимаешь ООП глубже, чем написано в методичке.
